## 12) CI/CD Overview (Jenkins + GitHub Webhooks + ngrok)


### What is CI/CD?
- **CI (Continuous Integration):** automatically builds and runs tests whenever code is pushed to the repository.
- **CD (Continuous Delivery/Deployment):** extends CI by automatically preparing releases/deployments (for this project we focus on CI + delivery of test results/reports).

In simple terms: **push code → auto-build → auto-run tests → save reports**.

### Why use GitHub Webhooks with Jenkins?
- **Jenkins** is the CI engine that runs your pipeline (install deps, install browsers, run Playwright tests, archive reports).
- **GitHub Webhooks** notify Jenkins instantly when you push/merge code.
- This reduces delays compared to only polling.

### How the automatic build trigger works
1. A developer **pushes code to GitHub**.
2. GitHub sends a **webhook HTTP request** to your **Jenkins endpoint** (via ngrok when Jenkins runs locally).
3. Jenkins receives the webhook event and **starts the pipeline**.
4. Jenkins runs the Playwright suite and **stores the HTML report**.

## 16) ngrok Setup (Expose local Jenkins)

When Jenkins runs locally, GitHub can’t reach `localhost`. ngrok fixes this by creating a public URL that forwards traffic to your local Jenkins.

### Step 1: Create/get an ngrok auth token
1. Create an ngrok account
2. Get your token from the ngrok dashboard

### Step 2: Authenticate ngrok
Run:
```bash
ngrok config add-authtoken YOUR_TOKEN
```

### Step 3: Expose Jenkins port 8080
Run ngrok (keep this terminal running):
```bash
ngrok http 8080
```

### What you should see
- ngrok will show a **forwarding URL** like:
  - `https://YOUR-NGROK-URL.ngrok-free.app`

### Key notes
- **Keep the ngrok terminal running** so the URL keeps working.
- Prefer the **HTTPS** forwarding URL (required for many webhook setups).

## 17) Jenkins URL Configuration (required for webhook callbacks)

After using ngrok, update Jenkins so it knows its external URL.

1. In Jenkins, go to: **Manage Jenkins → System → Jenkins URL**
2. Set the **Jenkins URL** to your ngrok HTTPS URL, typically ending like:
   - `https://YOUR-NGROK-URL`
3. Save

Why this matters: Jenkins and GitHub webhook/trigger plugins often need a correct public URL.

## 18) GitHub Webhook Setup

### Where to configure in GitHub
1. Open your GitHub repository
2. Go to **Settings → Webhooks → Add webhook**

### Webhook settings (recommended)
- **Payload URL**:
  - `https://YOUR-NGROK-URL/github-webhook/`
- **Content type**: `application/json`
- **Which events would you like to trigger this webhook?**:
  - Select **Just the push event**
- **Active** checkbox: **checked**

### Payload URL format
```text
https://YOUR-NGROK-URL/github-webhook/
```

## 19) Jenkins Job Configuration (Pipeline triggers)

### 1) Source Code Management → Git
In your Jenkins job (Pipeline):
1. Open the job
2. Go to **Source Code Management**

   - **Repository URL**: your GitHub repo URL

Enable webhook-based triggering:
1. Go to **Build Triggers**
2. Choose **GitHub hook trigger for GITScm polling**

Important: the plugin expects the GitHub webhook endpoint to match what GitHub sends and what Jenkins is configured to accept.

## 22) Troubleshooting

### 1) Webhook not triggering
**Possible causes**
- Wrong **Payload URL**
- ngrok stopped
- Jenkins not reachable from ngrok
- missing/incorrect Jenkins plugins

**Fixes**
- Ensure ngrok terminal is still running
- Verify Jenkins URL is set to your ngrok HTTPS URL
- Re-check Payload URL format:
  - `https://YOUR-NGROK-URL/github-webhook/`
- Confirm plugin installation and restart Jenkins

### 2) ngrok inactive
**Fix**
- Restart ngrok:
  - `ngrok config add-authtoken YOUR_TOKEN`
  - `ngrok http 8080`

### 3) localhost inaccessible (from GitHub)
GitHub can’t access `localhost`. You must use the ngrok public URL.

### 4) Wrong GitHub repo
If Jenkins points to a different repo/branch, it may build something unexpected.
**Fix**: verify Jenkins job → **Source Code Management → Repository URL**.

### 5) Missing plugins
If triggers/polling do not work, ensure these plugins exist:
- Git Plugin
- GitHub Plugin
- GitHub Integration Plugin
- Pipeline Plugin

Restart Jenkins after installation.

### 6) 404 webhook errors
**Cause**
- Jenkins endpoint path mismatch.

**Fix**
- Ensure webhook endpoint is exactly:
  - `/github-webhook/`
- Check if your Jenkins job/plugin expects a different path (rare).

### 7) Webhook delivery failures
**Fixes**
- Check GitHub webhook delivery logs (GitHub → Webhooks → Delivery)
- Verify Jenkins URL and ngrok HTTPS URL match what you configured
- Ensure Jenkins is running and accessible
